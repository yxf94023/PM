# 引言

1. 不要将有限的生命浪费在打造无人想要的产品上。
2. 你必须清楚地了解需要衡量那些指标以及如何进行衡量。 
3. 你最稀缺的资源是时间。在注定失败的项目上所花的每一分钟都是浪费。 因此， 对你来说最关键的就是快速、尽早并准确地识别出某个项目是否值得继续下去。此外， 你还需要向外部的利益相关者说明项目的进展。
4. 你自己就是你的商业创意的首位投资人。 你投资的是时间， 这比金钱还要宝贵。
5. 项目启动之后的道路， 不是变得更清晰， 而是更扑朔迷离。

## 淹没在数据的海洋中
1. 你并不需要海量的数据， 只需要有一些可行的指标足矣。
2. 指标的问题在于， 它们可以告诉你哪里不对， 但是无法告诉你症结所在。

## 专业化的诅咒
1. 虽然实验是一项关键活动， 但首先你得知道如何设计实验才能获得突破性的进展。
2. 如果我们定了一个新的指标， 在很长一段时间内这个指标是不具备指示意义的。
3. 很容易就会将相关性误认为因果关系。
4. 潮涨船高有时，墙倒众人推亦有时。

## 我们需要单一的衡量发展状况的指标
1. 在所有资源耗尽之前， 成功打造出可复制并可扩张的商业模式。 经验学习是这个过程中至关重要的一个环节， 但如果企业不能快速地将这种学习转化为可衡量的经营结果，那就只是徒增琐碎的经验罢了。

## 仅仅做实验是不够的
1. 科学家们先是创建一个模型， 然后再通过实验证明（或证伪）这个模型。

## 浪费无处不在
1. 所谓浪费， 就是消耗了资源却没有产生任何价值的人类活动。
2. 积跬步以致千里， 只要持续不断地进行微小的改进， 总生产力一定能获得巨大的提升。然而， 当面临变革创新的情境时， 问题就不再是找浪费， 而是按浪费的大小进行优先级排序。真正的挑战在于确定最关键的浪费行为， 剩下的则无线理会。
3. 策略的核心在于选择不做哪些事情。


## 面对一个系统
1. **过早优化陷阱（Premature Optimization Trap）**，对一个系统的各个环节进行加固是非常浪费的行为， 因为这样做并不能让整个系统整体更加坚固。 换而言之， 当我们试图去提升某个生产系统时， 只有正确地定位并聚焦于最弱的环节，才能获得最大的回报。
2. 当我们强化了某个环节后再次对系统各个环节施压时， 通常会发现新的最弱环节转移到了其他地方， 并且难以预测。 由此， 我们可以得出两个推论： 第一， 不停地强化某个最弱环节最终并不会产生任何收益， 因为其他环节早已取代它成为新的瓶颈， 限制了整条锁链的能力； 第二，由于我们无法预知新的瓶颈会转移到何处， 所以我们需要对整个系统进行持续监控， 不断地定位新的最弱环节。 盲目地加强单一环节， 即使它曾经是弱环节， 最后也会导致浪费。 这就是**局部优化陷阱（Local Optimization Trap）**。
3. 没有任何方法论能确保成功。 但是优秀的方法论可以为你提供一套反馈闭环， 让你不断学习， 持续进步。

## 第一章 市场认可度即目标
1. 站在不同的层级有不同看待事情的标准（或者成为目标）， 例如投资方没有必要去了解我们所知道的， 他们有自己的目标。 与解决方案本身相比， 他们更在乎的是能够在既定时间内为他们带来投资回报的商业模式。 下面才是他们真正想了解的。
+ 市场机会有多大？ 他们不在乎你的客户长什么样， 但是在乎你的客户有多少， 也就是**市场规模**。
+ 怎么挣钱？ 他们想知道的是你的成本结构和收入曲线， 也就是利润。
+ 他们还想知道， 如果你成功了， 你准备如何应对市场中不可避免的汹涌而来的模仿者和竞争者， 也就是你的**门槛优势**。

### 什么是市场认可度
1. 商业模式就是企业如何创造价值、传递价值和获得价值的基本过程。
2. 你的商业模式里不需要无法产生收益的业务。
3. 创造愉悦的客户与让客户感到愉悦是不同的。 让客户感到愉悦非常简单， 给他们提供免费的产品就可以了， 但这对商业模式来说是行不通的。 创造愉悦的客户不仅要让客户感到愉悦， 还要让他们在采纳你的解决方案时同样感觉愉悦。 结果才是最重要的。  减轻痛苦和提供安全感也可以称得上是某种愉悦。
4. 创造出愉悦或了不起的客户能让你获得回报。 商业的普世目标就是将这一过程重复地、可持续地发展下去。

### 有效产出即市场认可度
1. 客户有效产出率就是把非付费的用户转变为付费客户的比例。

### 商业模式原型
1. **直接型商业模式**， 是最基本、最普遍的商业模式之一。 因为用户都会被转化为客户， 所以该模式只有单一的行为主体。 在直接型商业模式中， 货币化价值的直接来源就是用户， 也就是后来的付费客户。这个简单的过程就是在客户生命周期中实现净收入的过程。
	单一行为主体的直接型商业模式还有其他例子：
	+ 零售
	+ SaaS
	+ 移动应用程序
	+ 实体商品
	+ 硬件
	+ 服务
2. **建立多方型商业模式**， 不同于直接型商业模式的将用户转化为客户的特点， 多方型商业模式是一个拥有多方行为主体的模型， 其中， 用户和客户是不同的主题（或者说“客户群细分”）。 多方型商业模式的商业目标依旧是创造价值、传递价值以及从用户哪里获得价值回报， 但是这个价值货币的过程是通过另外一个群里---客户来实现的。一般情况下， 用户不会直接使用现金货币为产品的使用买单， 而是通过某种衍生货币来进行。当产品得到足够多用户的认可之后， 这种衍生货币就代表了某些客户愿意通过购买而获得的一种衍生资产。
	+ 广告营利模式
	+ 大数据模式
	+ 企业软件
	+ 非营利模式
3. **建立平台型商业模式**， 其实是多方型商业模式的一种复杂的变体，在多方型商业模式中， 用户属于高风险一方， 用户的问题需要优先解决； 而平台模式则需要同时处理买方和卖方的问题。
4. 在平台型商业模式中， 市场认可度不是你创造买方或是卖方（上架量）的数量， 而是你把买卖双方撮合在一起进行交易的数量。

### 群组指标的不足
1. 你可以先使用聚合指标得出粗略的推断， 然后再使用群组指标来验证它
2. 撇开难以衡量不谈， 群组指标的一个更实际的不足是： 为了保证结果准确， 你不得不等待所有数据就位， 因为群组指标衡量的是实际的用户生命周期。
3. 解决这个问题的方法之一是使用更小单位的群组， 如每周甚至每天； 另一种解决之道是使用混合方案， 即综合使用聚合指标和群组指标。

    + 人力

    + 原材料
  + 策略瓶颈

    + 做事方式， 是由预先存在的流程和技术导致的
    + 衡量方式， 通常是由于采取了多种不同而且经常是相互冲突的衡量方式造成的

1. 当产品或服务的供给量超过市场对它的需求量时， 市场瓶颈就产生了。

6. 如果我们无法将做事方式与业务结果绑定起来， 就很难验证我们的做事方式是不是正确的。

>> 宏观指标可用于锁定热点的位置， 微观指标则可帮助你寻找原因。
>> 提供三个工具：（1）5Why分析法；（2）分析次要指标；（3）进行学习实验
>> 学习实验用于生成假设， 有效产出实验则用于验证假设

+ 利用约束，你总是能够找到瓶颈资源中挤出更多效率的方法， 或者在其他地方发现未被开发的潜能。 这一步骤通常只需很少资源甚至根本不用额外投入资源， 并且常常足以打破约束
+ 释放约束，一般来说， 释放约束需要增加被约束资源的数量。

### 认知偏差

#### 检验直觉时补偿自己的认知偏差的七大习惯
1. 预先宣布你预期的结果
>> 如果你只是漫无目的地想看看会发生些什么， 那么你一定会成功的， 因为总会有变化发生。
>> 聪明人都会为事情的合理性找各种理由， 创业者尤其精于此道
>> 一般来说， 企业不愿意预先宣布结果是因为以下两个更深层次的原因：（1）我们不愿证明自己是错误的；（2）我们很难对未知的事情作出有根据的猜测
2. 将个人成果变为团队结晶
>> 我在创业旅程中领悟到， 创造正确的产品远远比自己总是正确更加重要。这种心态的转变对树立健康的实验文化来说至关重要。
>> 只宣布保守的结果反映了你对这件事的恐惧。始终待在安全区内是不能实现突破的。 你需要建立一种企业文化： 只要可以被严格验证， 团队成员任何别出心裁的观点和天马行空的直觉都可以被接纳
>> 我建议你这样做：不要把宣布预期结果的重担压在一个人身上， 这应该是团队齐心协力的结晶； 过早地寻求团队共识可能会导致群体性思维。 预期结果特别容易受到团队中领导者意见的影响； 更好的做法是让团队成员先各自给出预期结果， 然后交换意见。
3. 注重预估过程， 而非精确结果
>> 你需要接受一个事实， 那就是你永远都无法获取充足的信息， 尽管如此， 你仍需要做出预测
>> 预测结果再不准， 也胜过没有预估。
4. 衡量行为， 而非反馈
5. 将你的假设转变为可伪证假设
>> 空泛的理论是无法被反驳的。
>> 可伪证在应对经验主义陷阱时是不可或缺的
6. 限制实验时间
>> 就像经验主义陷阱误导我们过早宣布成功一样， 缺乏时间限制将导致我们无限期地延长实验
7. 始终保持对照组
